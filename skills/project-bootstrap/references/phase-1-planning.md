## Steps

### 1. Project Overview Hearing

Ask the user:
- What kind of project do you want to create? (purpose, features, tech stack)
- Target users and use cases

### 2. Requirements Definition

Work with the user to define:
- **Functional requirements** (essential features, priorities)
- **Non-functional requirements** (performance, security, scalability)
- **Technology selection** with rationale

### 3. Project Basic Information

Finalize:
- Project name
- Short and long descriptions
- Programming language(s) and framework(s)

After requirements are gathered, create system specification documents and architecture diagrams as needed.

## Completion Criteria

Verify all of the following:
- [ ] Project overview is clearly defined
- [ ] Requirements definition is complete
- [ ] Project name and tech stack are decided

Report to user:
