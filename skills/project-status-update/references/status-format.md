# PROJECT_STATUS.md Format Reference

This document describes the format of `~/.ypm/PROJECT_STATUS.md` generated by the project-status-update skill.

## File Structure

```markdown
# Project Status

Last updated: YYYY-MM-DD HH:MM

## Active Projects

| Project | Overview | Branch | Last Updated | Phase | Progress | Next Task |
|---------|----------|--------|--------------|-------|----------|-----------|
| project-name | Short description | main | 2026-01-15 | Phase 2 | 60% | Implement feature X |
| project-name (Git worktree) | Description | feature/foo | 2026-01-14 | Phase 1 | 30% | Fix bug Y |

## In Development

| Project | Overview | Branch | Last Updated | Phase | Progress | Next Task |
|---------|----------|--------|--------------|-------|----------|-----------|

## Inactive Projects

| Project | Overview | Last Updated | Notes |
|---------|----------|--------------|-------|
```

## Section Categories

### Active Projects
- Projects updated within the configured active threshold (default: 7 days)
- Sorted by last update date (newest first)

### In Development
- Projects with recent activity but not yet in active use
- May have incomplete features or pending setup

### Inactive Projects
- Projects not updated within the active threshold
- Displayed with reduced detail (no branch/phase/progress columns)

## Column Descriptions

| Column | Description | Source |
|--------|-------------|--------|
| Project | Repository/directory name. Append "(Git worktree)" for worktree projects | scan_projects.py |
| Overview | Short project description | CLAUDE.md or README.md |
| Branch | Current git branch | git branch --show-current |
| Last Updated | Date of most recent git commit | git log -1 --format=%ci |
| Phase | Current development phase | CLAUDE.md |
| Progress | Implementation progress percentage | CLAUDE.md |
| Next Task | Next task to work on | GitHub Issues, CLAUDE.md, or commit messages |
| Notes | Additional context for inactive projects | Various sources |

## Language Handling

The output language is determined by `settings.language` in `~/.ypm/config.yml`:
- `en` (default): All headers and content in English
- `ja`: All headers and content translated to Japanese by Claude
